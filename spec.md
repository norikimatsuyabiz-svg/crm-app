# spec.md — ひとり営業用顧客管理アプリ 仕様書

## 1. 目的・利用シーン

### 目的

Excelの顧客台帳を置き換え、ひとりで動く営業担当が顧客情報と商談を一元管理できるブラウザアプリを提供する。業務用CRM SaaSの核機能（顧客管理・商談パイプライン）を最小構成で再現する。

### 利用シーン

- 新規顧客と名刺交換した直後に情報を登録する
- 商談前に顧客の過去メモ・商談履歴を確認する
- 週次で「見込み→提案→成約」のパイプライン全体を俯瞰し、次のアクションを決める
- 既存顧客への定期フォローアップメモを更新する

---

## 2. データ項目

### 顧客（Customer）

| フィールド名 | 型 | 必須 | 備考 |
|---|---|---|---|
| `id` | string | ○ | `"c_" + Date.now()` で自動採番 |
| `company` | string | ○ | 会社名（例: 株式会社サンプル） |
| `contactName` | string | ○ | 担当者名（例: 山田 太郎） |
| `title` | string | | 役職（任意） |
| `email` | string | | メールアドレス（任意） |
| `phone` | string | | 電話番号（任意） |
| `memo` | string | | メモ、複数行可（任意） |
| `createdAt` | string | ○ | ISO8601文字列、登録時に自動付与 |

localStorageキー: `crm-customers`（顧客オブジェクトの配列）

### 商談（Deal）

| フィールド名 | 型 | 必須 | 備考 |
|---|---|---|---|
| `id` | string | ○ | `"d_" + Date.now()` で自動採番 |
| `customerId` | string | ○ | 顧客の `id` を参照（1対多の外部キー） |
| `dealTitle` | string | ○ | 商談タイトル（例: サービスA導入提案） |
| `amount` | number \| null | | 金額・整数・円。未入力時は null |
| `status` | string | ○ | `"lead"` / `"proposal"` / `"won"` の3値 |
| `followUpMemo` | string | | フォローアップメモ、複数行可（任意） |
| `createdAt` | string | ○ | ISO8601文字列、登録時に自動付与 |
| `updatedAt` | string | ○ | ISO8601文字列、保存のたびに更新 |

localStorageキー: `crm-deals`（商談オブジェクトの配列）

### 1対多の関係

```
顧客 (Customer) 1 ──< 商談 (Deal) * 
       id ←──────────── customerId
```

1件の顧客に対して、0件以上の商談が紐付く。商談側が `customerId` を持つ（顧客側は参照を持たない）。

## 3. 画面構成

### 全体レイアウト

```
┌──────────────────────────────────────────────────────┐
│ タブ: [顧客]  [パイプライン]                          │
├──────────────────────────────────────────────────────┤
│ ビュー①顧客  or  ビュー②パイプライン                 │
└──────────────────────────────────────────────────────┘
```

### ビュー① 顧客（ツーカラム）

```
┌─────────────────────┬────────────────────────────────┐
│ 左ペイン (340px)    │ 右ペイン (flex-1)              │
│                     │                                │
│ [検索ボックス]      │  モード切替（4種）:            │
│ [新規顧客ボタン]    │  - 空状態（初期表示）          │
│                     │  - 顧客詳細                    │
│ 顧客カード ×N       │  - 顧客フォーム（新規・編集）  │
│ (クリックで選択)    │  - 商談フォーム（新規・編集）  │
│                     │                                │
└─────────────────────┴────────────────────────────────┘
```

#### 左ペインの仕様

- 検索ボックス: 会社名・担当者名・役職を横断インクリメンタル検索
- 顧客カードの並び順: `createdAt` の新しい順
- 選択中のカード: アクセントカラーでハイライト

#### 右ペインの各モード

| モード | 表示内容 | 遷移元 |
|---|---|---|
| 空状態 | 「顧客を選択してください」等の案内 | 起動時・顧客削除後 |
| 顧客詳細 | 連絡先ブロック＋商談一覧＋「商談を追加」ボタン＋編集・削除ボタン | 顧客カードのクリック・保存後 |
| 顧客フォーム | 顧客の入力フォーム（新規時は空、編集時は既存値入り） | 「新規顧客」ボタン・詳細の編集ボタン |
| 商談フォーム | 商談の入力フォーム（新規時は空、編集時は既存値入り）＋削除ボタン | 「商談を追加」ボタン・商談行のクリック |

### ビュー② パイプライン（カンバン3列）

```
┌────────────┬────────────┬────────────┐
│  見込み    │  提案      │  成約      │
│  (lead)    │ (proposal) │  (won)     │
│            │            │            │
│ [商談カード│ [商談カード│ [商談カード│
│  会社名    │  会社名    │  会社名    │
│  金額  →] │ ← 金額  →]│ ← 金額   ]│
└────────────┴────────────┴────────────┘
```

#### パイプラインカードの仕様

- 表示内容: 商談タイトル・紐付く顧客の会社名・金額
- 「←」ボタン: 1つ前のステータスへ移動（先頭列 `lead` では非表示）
- 「→」ボタン: 1つ後のステータスへ移動（末尾列 `won` では非表示）
- カードをクリック: 顧客ビューに切り替え、該当顧客の商談フォーム（編集）を開く
- ステータス移動は即時保存（確認ダイアログなし）

## 4. CRUD操作フロー

### 顧客のCRUD

| 操作 | トリガー | 遷移フロー |
|---|---|---|
| Create | 「新規顧客」ボタン | 顧客フォーム（空） → 保存 → 顧客詳細 |
| Read | 起動時・検索 | 左ペインに一覧表示、カードクリックで顧客詳細 |
| Update | 顧客詳細の「編集」ボタン | 顧客フォーム（既存値入り） → 保存 → 顧客詳細 |
| Delete | 顧客詳細の「削除」ボタン | 確認ダイアログ → 顧客＋紐付く商談を削除 → 空状態 |

### 商談のCRUD

| 操作 | トリガー | 遷移フロー |
|---|---|---|
| Create | 顧客詳細の「商談を追加」ボタン | 商談フォーム（空） → 保存 → 顧客詳細（商談一覧に反映） |
| Read | 顧客詳細の商談一覧 / パイプラインビュー | 一覧表示（ステータスバッジ付き） |
| Update | 商談行のクリック / パイプラインカードのクリック | 商談フォーム（既存値入り） → 保存 → 顧客詳細 |
| Delete | 商談フォーム内の「削除」ボタン | 確認ダイアログ → 商談削除 → 顧客詳細 |
| ステータス遷移 | パイプラインの「←」「→」ボタン | 即時ステータス更新・保存（フォームを経由しない） |

## 5. 永続化方針

### localStorageキーと保存形式

| キー | 形式 | 説明 |
|---|---|---|
| `crm-customers` | JSON文字列（配列） | 顧客オブジェクトの配列 |
| `crm-deals` | JSON文字列（配列） | 商談オブジェクトの配列 |

```js
// 読み込み
const customers = JSON.parse(localStorage.getItem('crm-customers') || '[]');

// 書き込み
localStorage.setItem('crm-customers', JSON.stringify(customers));
```

### 顧客削除時の連鎖削除ルール

顧客を削除する際は以下を**同時に**実行する（片方だけ削除しない）：

1. `crm-customers` から対象顧客を削除
2. `crm-deals` から `customerId === 削除顧客のid` のすべての商談を削除

## 6. 初期データ

### 投入条件

- アプリ起動時に `crm-customers` キーが存在しない（または空配列）の場合のみ投入する。
- 一度でも顧客が登録されていれば投入しない。

### 初期データの内容

顧客3件・商談5件を自動投入する。商談は3つのステータス（`lead` / `proposal` / `won`）に分散させる。会社名・人物名は実在しないものを使用する。

**顧客（例）:**

| 会社名 | 担当者名 | 役職 |
|---|---|---|
| 株式会社テラノバ | 北村 恵 | 購買部長 |
| ミドリ工業株式会社 | 坂本 龍一郎 | 代表取締役 |
| フォレスト・パートナーズ合同会社 | 橘 みなみ | 事業開発マネージャー |

**商談（例）:**

| 商談タイトル | 顧客 | ステータス | 金額 |
|---|---|---|---|
| クラウド移行支援 | 株式会社テラノバ | won | 1,200,000 |
| 業務効率化ツール導入 | 株式会社テラノバ | proposal | 480,000 |
| 製造ライン最適化コンサル | ミドリ工業株式会社 | proposal | 980,000 |
| 新規事業調査レポート | フォレスト・パートナーズ合同会社 | lead | null |
| マーケティング支援契約 | フォレスト・パートナーズ合同会社 | lead | 250,000 |

## 7. 対象外の機能

以下は本アプリの対象外とする。将来の拡張候補として記録する。

| 機能 | 理由・代替手段 |
|---|---|
| ドラッグ＆ドロップでのステータス移動 | 「←」「→」ボタンクリックで代替 |
| クラウド保存・他端末共有 | localStorage のみ。エクスポートで代替検討余地あり |
| ログイン・マルチユーザー | ひとり営業用途のため不要 |
| CSVインポート・エクスポート | 初期リリース対象外 |
| モバイル対応（レスポンシブ） | PC表示のみ |
| メール送信連携 | 外部API禁止 |
| カレンダー連携 | 外部API禁止 |
| 通知・リマインダー | 初期リリース対象外 |
| レポート・集計グラフ | 初期リリース対象外 |
